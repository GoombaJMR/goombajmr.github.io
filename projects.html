Rotor Router

JUN 2018 - AUG 2018

When we send out a little memoryless bot to go explore a graph, how do we give the bot instructions so that it can perpetually navigate through every vertex and edge in a graph effectively? The rotor router model seemed to be a great solution, and had been researched to some extent in regards to lower, upper bounds and flaws, but little mention of real experiments and simulations performed on the model with actual graphs. I decided to create the first simulator for the rotor router algorithm that specifically looks for the average stabilization period of a mobile bot in a graph. In layman's terms, the bot will magically start traveling through a path in the graph that will loop forever. We want to find what this path (is it AB->BD->DA->AB, or something else, for example?) and how long it takes to start on that path - the stabilization period (counted in steps, or movements by the bot). You can read more about my analysis of my simulation results and improvements to the original rotor router mechanism in my paper here.
